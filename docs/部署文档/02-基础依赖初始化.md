# 基础依赖初始化

本文档说明 Ubuntu 上如何准备本项目依赖的基础服务：

- MySQL：主业务数据库
- MinIO：图片与 APK 对象存储
- Qdrant：向量检索服务

> 但是我更推荐你直接用`docker`启动，这样只用管`docker`就可以了

## 1. MySQL 初始化

### 1.1 安装 MySQL

在 Ubuntu 上安装：

```bash
sudo apt update
sudo apt install -y mysql-server
```

启动并设置开机自启：

```bash
sudo systemctl enable --now mysql
sudo systemctl status mysql
```

### 1.2 基础安全配置

建议至少为 `root` 设置密码，并限制远程访问。

进入 MySQL：

```bash
sudo mysql
```

然后执行：

```sql
ALTER USER 'root'@'localhost' IDENTIFIED WITH caching_sha2_password BY 'replace-with-strong-password';
FLUSH PRIVILEGES;
```

### 1.3 创建业务用户

建议不要让后端直接使用 `root`：

```sql
CREATE USER 'mathnotebook'@'127.0.0.1' IDENTIFIED BY 'replace-with-strong-password';
GRANT ALL PRIVILEGES ON wrong_question_book.* TO 'mathnotebook'@'127.0.0.1';
FLUSH PRIVILEGES;
```

如果你的后端通过 `localhost` 连接，也可以额外加：

```sql
CREATE USER 'mathnotebook'@'localhost' IDENTIFIED BY 'replace-with-strong-password';
GRANT ALL PRIVILEGES ON wrong_question_book.* TO 'mathnotebook'@'localhost';
FLUSH PRIVILEGES;
```

### 1.4 导入初始化表结构

项目当前的初始化 SQL 位于：

- [deployments/mysql/mysql.sql](../../deployments/mysql/mysql.sql)

导入方式：

```bash
mysql -u root -p < deployments/mysql/mysql.sql
```

导入后验证：

```bash
mysql -u root -p -e "USE wrong_question_book; SHOW TABLES;"
```

你应该至少看到这些表：

- `user`
- `file_record`
- `wrong_question`
- `tag`
- `wrong_question_tag`
- `question_vector`
- `ocr_record`
- `ai_analysis_record`
- `review_record`

## 2. MinIO 初始化

### 2.1 作用说明

MinIO 在本项目里承担两类文件：

- 错题原图
- Android APK 发布包

后端配置中与 MinIO 相关的字段位于：

- [backend/configs/config.example.yaml](../../backend/configs/config.example.yaml)

### 2.2 准备目录

```bash
sudo mkdir -p /opt/minio/bin
sudo mkdir -p /opt/minio/data
sudo mkdir -p /opt/minio/config
sudo chown -R minio-user:minio-user /opt/minio
```

### 2.3 安装 MinIO 二进制

建议从 MinIO 官方下载 Linux 二进制并放到：

```text
/opt/minio/bin/minio
```

下载完成后赋予可执行权限：

```bash
sudo chmod +x /opt/minio/bin/minio
```

### 2.4 写入 systemd 服务

仓库已提供模板：

- [deployments/systemd/minio.service.example](../../deployments/systemd/minio.service.example)

复制到系统目录：

```bash
sudo cp deployments/systemd/minio.service.example /etc/systemd/system/minio.service
```

按实际情况修改：

- `MINIO_ROOT_USER`
- `MINIO_ROOT_PASSWORD`
- `--address`
- `--console-address`

然后启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now minio
sudo systemctl status minio
```

### 2.5 创建 bucket

本项目默认 bucket 名为：

```text
wrong-question-images
```

你可以通过 MinIO Console 创建，也可以用 `mc` 创建。

项目中后端会在启动时尝试自动建 bucket，但生产上仍建议你提前手动确认。

### 2.6 目录约定

当前项目里有两个重要路径约定：

- 错题图片：业务代码自动生成 object key
- Android APK：必须上传到

```text
{bucket}/apk/
```

例如：

```text
wrong-question-images/apk/math-notebook.apk
```

### 2.7 MinIO 生产建议

- 不要继续使用 `change-me`
- 如果对外开放对象访问，优先走独立域名或反代域名
- 尽量使用 HTTPS
- 仅在确认业务需要时启用公共读

## 3. Qdrant 初始化

### 3.1 作用说明

Qdrant 用于存储：

- `semantic` 向量
- `mistake` 向量

后端配置项：

```yaml
vector:
  collection_name: wrong_question_vectors
  qdrant_url: http://127.0.0.1:6333
```

### 3.2 部署建议

Qdrant 是本项目的重要依赖，但它的部署方式可以灵活选择：

- 单机自托管
- Docker 容器
- 外部托管实例

如果你的目标是尽快跑通业务链路，推荐先准备一个稳定可访问的 Qdrant 实例，然后把地址写入后端配置即可。

### 3.3 最小验证要求

无论你用哪种方式部署，至少要满足：

- 后端机器能访问 `QDRANT_URL`
- 端口连通
- 后端启动时可自动创建 collection

### 3.4 网络与安全建议

- Qdrant 端口不要直接暴露公网
- 如启用了 API Key，记得同步配置 `qdrant.api_key`
- 建议只允许后端所在主机访问

## 4. 依赖初始化后的联通性检查

### MySQL

```bash
mysql -u mathnotebook -p -h 127.0.0.1 -e "USE wrong_question_book; SHOW TABLES;"
```

### MinIO

检查浏览器是否能打开 Console，或确认对象上传可用。

### Qdrant

确认它的健康接口或 collection API 能正常访问。
